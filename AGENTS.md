## Architecture

**Stack**: TanStack Start + TanStack Router (file-based) + React 19 + Tailwind CSS v4 + oRPC + Valibot + Biome + Nitro + Pulumi (Docker)

**Aspen OS integration**: This app depends on `@aspen-os/*` packages (`framework`, `organization`, etc.) from the sibling monorepo at `/home/aditya/projects/aspen-os`. Vite's `server.fs.allow` and `opencode.json` both grant access to that directory. The framework provides auth, database, storage, and module system.

**Key directories**:
- `src/aspen/` — Aspen Framework integration: `server.ts` (server-side init: DB, auth, storage), `client.ts` (client-side init), `auth.ts` (access control + roles: admin, doctor, receptionist)
- `src/rpc/` — API layer: `procedures/` (oRPC procedure implementations), `router.ts` (aggregates all procedures), `schema.ts` (Valibot schemas)
- `src/lib/rpc.client.ts` — isomorphic oRPC client with TanStack Query utils
- `src/routes/` — file-based routes; `(protected)/` is the authenticated route group containing CRM, EMR, OPD, pharmacy, HR, settings, tasks, and ERP modules
- `src/integrations/` — PostHog provider, TanStack Query setup
- `infra/` — Pulumi IaC: `docker/` has app + postgres container definitions

**API routes** (`src/routes/api/`):
- `rpc.$.ts` — oRPC RPC handler at `/api/rpc`
- `$.ts` — OpenAPI handler at `/api` with Swagger docs
- `auth.$.ts` — Auth handler (delegates to `f.auth.fetch_handler`)

**Adding a new oRPC procedure**: define it in `src/rpc/procedures/`, add to the router object in `src/rpc/router.ts`, add schema in `src/rpc/schema.ts`.

**Validation library is Valibot** (not Zod). Schemas in `src/rpc/schema.ts` use valibot's `object()`, `string()`, `pipe()`, etc. The OpenAPI handler uses `ZodToJsonSchemaConverter` from `@orpc/zod/zod4` for spec generation despite schemas being Valibot — both libraries are installed.

## Generated files — do NOT edit

- `src/routeTree.gen.ts` — auto-generated by TanStack Router CLI. Biome excludes it. VS Code marks it read-only.
- Regenerate with `bun run gen:routes` after adding/removing route files.

## Code style

- **Biome**: 2-space indent, double quotes, semicolons, line width 80
- Run `bun run check:lint` before committing (auto-fixes lint + format)
- `useSortedClasses` Biome rule is enabled — Tailwind class order is auto-sorted in `className`, `classList`, and calls to `clsx`/`cva`/`tw`
- Linter is **disabled** for `src/components/ui/**` (shadcn components)

## Path aliases

`@/*` resolves to `./src/*` (configured in `tsconfig.json` paths, resolved by Vite via `tsconfigPaths`).

## Environment variables

Type-safe via T3Env in `src/env.ts`. Client vars must be prefixed `PUBLIC_` (not `VITE_` — `vite.config.ts` sets `envPrefix: "PUBLIC_"`). Server vars have no prefix. Add new vars to `src/env.ts` with Zod schemas.

## UI components

Shadcn (`base-nova` style, zinc base color, lucide icons). Install with:
```bash
bunx shadcn@latest add <component>
```

Component aliases: `@/components/ui`, `@/lib/utils`, `@/hooks`.

## React Compiler

React Compiler is enabled via `babel-plugin-react-compiler` (through `@rolldown/plugin-babel` in `vite.config.ts`). Components are auto-memoized — avoid manual `useMemo`/`useCallback`/`memo`.

## Deployment

Docker image builds with `oven/bun:1`, runs all `gen:*` scripts before `vite build`, then serves with `bun run dist/server/index.mjs` on port 3000.

Pulumi manages Docker infrastructure (app container + PostgreSQL 18) with stacks: `dev`, `production`, `preview`. Config is in `Pulumi.<stack>.yaml`. The `dev` stack runs the dev server via a dynamic resource instead of building a Docker image.
