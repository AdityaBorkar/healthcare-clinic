# AGENTS.md

## Commands

| Task | Command |
|------|---------|
| Install deps | `bun install` |
| Dev server (port 3000) | `bun --bun run dev` |
| Build | `bun --bun run build` |
| Lint | `bun --bun run lint` |
| Format | `bun --bun run format` |
| Check (lint + format) | `bun --bun run check` |
| Test | `bun --bun run test` |
| Generate routes | `bun --bun run generate-routes` |

Always use `bun`, not `pnpm` or `npm`. The `.cursorrules` file incorrectly references `pnpm`.

## Architecture

**Stack**: TanStack Start + TanStack Router (file-based) + React 19 + Tailwind CSS v4 + oRPC + Biome + Nitro

**Key directories**:
- `src/routes/` — file-based routes; each file exports a `Route` via `createFileRoute`
- `src/orpc/` — API layer: `router/` has procedure implementations, `schema.ts` has Zod schemas, `client.ts` is the isomorphic oRPC client
- `src/integrations/` — PostHog provider, TanStack Query setup
- `src/lib/` — utilities (`cn()` for Tailwind class merging)

**API routes**:
- `src/routes/api.rpc.$.ts` — oRPC RPC handler at `/api/rpc`
- `src/routes/api.$.ts` — OpenAPI handler at `/api` with Swagger docs

**Adding a new oRPC procedure**: define it in `src/orpc/router/`, add to `src/orpc/router/index.ts` exports, add schema in `src/orpc/schema.ts`.

## Generated files — do NOT edit

- `src/routeTree.gen.ts` — auto-generated by TanStack Router CLI. Biome excludes it. VS Code marks it read-only.
- Regenerate with `bun --bun run generate-routes` after adding/removing route files.

## Code style

- **Formatter**: Biome with tabs, double quotes
- **Linter**: Biome recommended rules
- Run `bun --bun run check` before committing (covers lint + format)
- Biome scope: `src/**`, `.vscode/**`, `index.html`, `vite.config.ts` — excludes `routeTree.gen.ts` and `styles.css`

## Path aliases

Both `@/*` and `@/*` resolve to `./src/*`. Prefer `@/*` (it's in `package.json` imports and used in existing code).

## Environment variables

Type-safe via T3Env in `src/env.ts`. Client vars must be prefixed `VITE_`. Add new vars to `src/env.ts` with Zod schemas.

## UI components

Shadcn (new-york style). Install with:
```bash
bunx shadcn@latest add <component>
```
(Not `pnpm dlx` — the `.cursorrules` is wrong.)

Component aliases: `@/components/ui`, `@/lib/utils`, `@/hooks`.

## React Compiler

React Compiler is enabled via `babel-plugin-react-compiler` in `vite.config.ts`. Components are auto-memoized — avoid manual `useMemo`/`useCallback`/`memo`.

## Deployment

Nitro builds a self-contained Node server:
```bash
bun --bun run build
node dist/server/index.mjs
```
